Purchase Request System - HOME

Description

The Purchase Request System (PRS) is an application that provides users with the ability to request items they need. Each request can contain multiple different lines that specify the product and the quantity of that product.

Every user is required to log into the system with a user name and password. Users can be identified optionally as reviewers and/or administrators.

Any user can create a purchase request, add products to that request, and mark the purchase request ready for review. Once set to review, a user identified as a reviewer may review the purchase request and either accept or reject the purchase request. If rejected, the original user can revise the purchase request and set it to be reviewed again. A reviewer may not review their own purchase requests.

Only users identified as administrators may maintain the users, vendors, and products.

To satisfy the approved purchase request, the system provides capabilities to create purchase orders to vendors. This function selects a vendor from the list of vendors and reviews the approved purchase requests that include products from that vendor. The total number of each product approved for purchase request is added to the purchase order. After all the vendor's products to be purchased are included on the purchase order, the price and quantity is used to create a line item total and all lines are added to create a subtotal. Then a ten percent (10%) tax is added based on the subtotal along with a five percent (5%) shipping fee based on the subtotal to calculate the final total of the purchase order.
